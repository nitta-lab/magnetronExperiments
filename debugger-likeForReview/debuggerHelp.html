How to operate the debugger-like view

With respect to the real-world programs, we extract only the source code related to each subject. The extracted source code of a real-world program is presented on the website in a Eclipseâ€™s debugger-like view. The debugger-like view allows the participants to control the program execution flow stepping through the source code.
First, press the debug button to start debugging.

The debugger-like view is displayed on the left, and the running line is highlighted in light green.

If you press the second step into button from the left here, you will move to the called destination, and the called destination method will also be stacked on the stack on the debugger-like view.

The description of each button is as follows:


  • If you press the leftmost button, debug execution will resume from the beginning even during debug execution.
  • Press the second button from the left to step over to the called method,
  • Press the third button from the left to step into the next line in the same method,
  • Press the rightmost button to step return to the calling method.
Note that the website's back button may not work properly on the debugger-like view.
